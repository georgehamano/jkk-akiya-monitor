## 1. システム構成

```
[定期実行: Colab / GitHub Actions]
        │
        ▼
[jkk_line_notify.py]
   ├─ HTTP Session（ウォームアップ → 監視URL）
```

## 2. URL・リクエスト仕様

### 2.1 ウォームアップ（順序固定推奨）

1. `HOME_URL`  
   - 既定: `https://www.to-kousya.or.jp/`
2. `CHINTAI_URL`  
   - 既定: `https://www.to-kousya.or.jp/chintai/index.html`
3. `JH_WARMUP_URL`（環境変数 `JKK_JHOMES_WARMUP_URL` で上書き、空でスキップ可）  
   - 既定: `https://jhomes.to-kousya.or.jp/search/jkknet/service/akiyaJyoukenStartInit`
4. **禁止**: `https://jhomes.to-kousya.or.jp/`（404）

### 2.2 監視対象（一覧取得）

- 環境変数: `JKK_TARGET_URL`
- 既定（要件上の推奨）:  
  `https://jhomes.to-kousya.or.jp/search/jkknet/service/akiyaJyokenDirect`  
  （検索結果相当。`AKIYAchangeCount` は別用途・おわび表示になりやすい場合あり）
- 監視 URL 取得時の `Referer` 推奨: `CHINTAI_URL`

### 2.3 HTTP ヘッダ（最低限）

| ヘッダ | 例・備考 |
|--------|----------|
| User-Agent | 最新 Chrome 相当の文字列 |
| Accept | `text/html,application/xhtml+xml,...` |
| Accept-Language | `ja,en-US;q=0.9,...` |
| Referer | ウォームアップ・直前ページに合わせる |
| Cache-Control / Pragma | `no-cache` 推奨 |

### 2.4 レスポンスのデコード

- `Response.content` をバイトのまま扱い、試行順:  
  `apparent_encoding` → `cp932` → `shift_jis` → `euc_jp` → `utf-8`
- 次のいずれかを含むデコード結果を優先:  
  `住宅名`, `募集戸数`, `cell666666`, `senPage`, `物件名`, `空き家`
- **注意**: サービス名文字列（`jkknet`, `akiyaJyokenDirect` 等）だけでは一覧判定に使わない（中間ページに誤マッチするため）。

## 3. 中間ページ・遷移仕様

### 3.1 中間ページの検知

以下のいずれかを `html.unescape` 後の本文に含む場合、中間ページとみなす:

- `数秒後に自動で次の画面`
- `しばらくたっても表示されない場合`
- `自動で次の画面`

### 3.2 遷移先 URL の抽出（候補）

優先的に以下から URL を収集（重複除去、`urljoin(base_url, ...)` で相対解決）:

1. `<meta http-equiv="refresh" content="...;url=...">`
2. `<a href>` かつリンク文言に `こちら` / `クリック` / `次の画面`
3. `href="#"` でも `onclick` 内に `http(s)://` や `location.href='...'` 等
4. `<script>` 内の `location.href` / `window.location` / `location.replace`
5. 本文中の `https?://...` および `/search/jkknet/...`

### 3.3 候補のフィルタ

除外推奨:

- `/images/`, `.gif`, `.css`, `.js`, `.png`, `.jpg` 等

優先推奨:

- `search/jkknet/` かつ `.jsp` または `service/` または `view` または `result`

### 3.4 ループ防止

- 訪問済み URL 集合 `visited` を保持し、`norm(url)=rstrip('/')` で同一化。
- **現在ページ URL と同一の候補は試行しない**。
- 再帰深度 `max_depth`（例: 3）で打ち切り。

### 3.5 フォームフォールバック

候補 URL が空、または有効な遷移が得られない場合:

- HTML 内の `<form>` を最大 3件まで試行。
- `method` 既定は `get`。`input name/value` を収集。
- `action` 空なら `base_url` に対して送信。
- 返却 HTML で `parse_properties` が非空なら成功。

### 3.6 `get_with_session` の戻り値

- 1回目の `TARGET_URL` 応答を `html1` とする。
- `maybe_follow_transition(session, res.url, html1)` が HTML を返せばそれを採用、否则 `html1`。

## 4. パース仕様（一覧表）

### 4.1 対象テーブル

- セレクタ: `table.cell666666` を優先。無ければ全 `table` を走査。

### 4.2 ヘッダ行の決定

- 先頭から最大 6 行を走査し、セル文字列から列インデックスを決定:
  - **住宅名列**: `物件名` または `住宅名` を含む
  - **サブ列（行識別）**: `号室` または `間取り`（号室優先）
  - **件数列**: `募集戸数` または 空き家・空室系

### 4.3 データ行の1レコード

| フィールド | 内容 |
|------------|------|
| name | 住宅名（文字列、前後空白除去） |
| room | 間取りまたは号室（無ければ `-`） |
| count | `募集戸数` セルから最初の整数 |
| detail_url | `build_detail_url_from_row` の結果 |
| senpage | `senPage` 4引数を `,` 連結（無ければ空） |

### 4.4 固定列フォールバック

ヘッダ認識失敗時:

- 列 1: 住宅名  
- 列 5: 間取り  
- 列 9: 募集戸数  
- ヘッダ行除外: 住宅名セルが文字列 `住宅名` の行

### 4.5 `senPage` 正規表現

- シングルクォート版・ダブルクォート版の両方をサポート。

## 5. 集計・差分仕様

### 5.1 `last_data.json`

- 形式: JSON オブジェクト  
  `{ "住宅名A": 合算件数, "住宅名B": 合算件数, ... }`
- 同一 `name` の複数行は **募集戸数の合算**。

### 5.2 `last_rooms.json`（入れ替わり用）

- 形式: JSON オブジェクト  
  `{ "住宅名A": "sha256hex", ... }`
- 各住宅名について、行ごとのシグネチャ  
  `f"{room}|{count}|{senpage}"` をソート結合し SHA256。

### 5.3 通知判定

| 条件 | reason 値 | increase | current_total |
|------|-----------|----------|----------------|
| 新規物件（前回辞書に無し） | `new` | 今回合算 | 今回合算 |
| 合算件数増加 | `increase` | 差分 | 今回合算 |
| 合算同一・シグネチャ変化 | `rotation` | 0 | 今回合算 |

### 5.4 状態保存タイミング

- **通知成否に関わらず**、今回の `current_map` と `current_fp` を保存する（次回比較用）。

## 6. 詳細 URL 仕様

### 6.1 テンプレート解決順

1. `JKK_DETAIL_URL_TEMPLATE`（完全 URL、`{p1}`〜`{p4}`）
2. `JKK_DETAIL_VIEW_BASE` + `JKK_DETAIL_QUERY_TEMPLATE`
3. 内蔵デフォルト:  
   - base: `https://jhomes.to-kousya.or.jp/search/jkknet/view`  
   - query: `danchi={p3}&room={p2}`

### 6.2 senPage 引数の割当（既定）

| 引数 | 例 | 既定クエリでの意味 |
|------|-----|-------------------|
| p1 | 空 | テンプレートで任意 |
| p2 | L8851 | `room` |
| p3 | 1280950 | `danchi` |
| p4 | 0000 | テンプレートで任意 |

## 7. LINE Messaging API 仕様

### 7.1 エンドポイント

- `POST https://api.line.me/v2/bot/message/push`

### 7.2 環境変数

| 変数 | 必須 | 説明 |
|------|------|------|
| `LINE_CHANNEL_ACCESS_TOKEN` | 通知時は必須 | チャネルアクセストークン |
| `LINE_TO` | 通知時は必須 | userId / groupId / roomId |

未設定: 通知スキップ、`[INFO]` で明示。

### 7.3 メッセージ本文（1物件あたり）

```
【JKK 空き家状況更新】
物件名: {name}
増えた件数: +{increase} {理由補足}
現在の総数: {current_total}
詳細URL: {detail_url}
```

- 1リクエストあたり最大 5 メッセージ（LINE仕様）。超過分は分割 POST。
